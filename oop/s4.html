Flow control and looping

Most of the time ua tunataka program ifanye execution kulingana na condition flani, mfno, endapo ikakuta namba hii ni sawa na ile bhas ifanye kitu flani. Au tunataka program ifanye kazi kulingana na idadi ya vitu flan. Kutimiza ivo tunatumia constructs mbalimbali.

Flow control constructs

  1. if ...
  2. else if ...
  3. if...else

The "if ..." construct (Syntax)

if ... ua inaexecute kulingana na condition moja, kama io condition ikiwa "true" bhac statements zilizondani ya io if ztakua execute, otherwise hakutakua na output.

if(condition) {

//statements

}
 Mfano:
if(7 < 9) {
System.out.println("7 ni ndogo kuliko 9");
}
 Ukicheki io condition ndani ya if utaona ni true, saba ni ndogo kuliko tisa, so io statement ilio ndani ya { }, itakua executed.
OUTPUT
7 ni ndogo kuliko 9
very simple....

The "else if ..." construct.

else if ... ua tunaitumia kama una more than one condition.

if (7 < 9){
System.out.println("7 ni ndogo kuliko 9");
}
else if ( 4 > 2){
System.out.println("4 ni kubwa kuliko 2");
}

Ukichek utaona kuna condition mbili, kama zote ni true bhac statements zote ztakua executed.

The "if else ..." construct.

if else ... ua tunaitumia kama una otherwise... Mfno 'if umri ni mdogo chini ya miaka 18 ingia disco toto, otherwise (means umr mkubwa juu ya 18) ingia Casino'. Ilo neno "otherwise" ndo "else" ndani ya java.

int age = 19;
if (age < 18){
System.out.println("Ingia disco toto.");
}
else{
System.out.println("Ingia kwe Casino watoto wazuri wakufilisi");
}

Apo kwanza nimestore '19' kwe variable inaitwa 'age'. Then nikaitest ndani ya "if".
Ikikutwa kweli namba ilio ndani ya variable "age" ni ndogo kuliko 18, bac output itakua Ingia disco toto
Kama ni kubwa kulko 18, means condition imekua ya uongo, itaacha kuexcute statement ya kwanza na itaexecute statement ilio ndani ya "else".
So, Output itakua Ingia kwe Casino watoto wazuri wakufilisi
Beautiful...... So far so good.
