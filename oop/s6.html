Classes and objects.

Hapa ndipo penye hekima!

Asilimia 99 ya java programs ni lazima zidepend on the manipulation of classes and objects.

Bila kupoteza time, nini maana ya class na nini maana ya object....

Tunasemaga A class is a blueprint of an object. Short and clear.

Pia tunasemaga An object is an instance of a class
Ili utengeneze object lazima ue na class. Na class ndo inakupa mwongozo ni aina gani ya object unataka kutengeneza

Hua tunatumia neno new kuunda object.
Kitendo cha kutengeneza object kinaitwa Instantiation (Creating an instance of a class).

So, how do we create a class?

Kwanza inabidi ujue ni variables zipi zinaitajika kutunza data, na method zipi nahitaji ili kuchakata data zako.
Variables and methods ndani ya class kwa ujumla tunaziitaga data members (Instance variables).

Sometimes utataka data flani isiweze kuonekana wala kuchakatwa nje ya class, au unaeza taka data flani ndani ya class iweze kuonekana hata nje ya class husika, Kitendo hiki cha kuficha baadhi ya data na kuonyesha baadhi ya data tunakiita Encapsulation.
Ili kuficha data ua tunatumia neno private Na ili kuzifanya zionekane tunatumia neno public kwenye variable and method definitions

Tuone mfano mmoja wa class.

class Animal {

private double weight;
private int legs;
public String color;

}

Apo variables zote tulizoziunda kwa neno private zitaonekana tu ndani ya class Animal, Apa namanisha huezi zipata nje ya class Animal.
Na Variable ambayo tumeiunda kwa kutumia neno public itaweza kutumika na kuonekana nje ya class

Accessor and mutator

In normal language, an Accessor is a thing that have given a permission to get something, In swahili we say that an accessor ni mtu au kitu ambcho kina ruusa ya kupata au kuchukua kitu kutoka seem ambyo mtu mwingne ambye hana ruusa hawez chukua au kupata.

But a mutator is a thing that can change something.

In the above class tumeona kuna variables ambazo ni 'private' na nyingne ni 'public', na tukasema kwamba 'private' data aziwezi onekana nje ya class, so how can we get and change them when we need to?
The answer is simple, we use Accessors and mutators

So how do we do?

Ili kuweza zipata 'private' data nje ya class utatumia Accessors, na kama unataka kubadili data iliomo ndani ya memory utatumia Mutators
Look:

public class Animal {

private int legs;
private double weight;

//Accessors
public int getLegs(){
return legs;
}

public double getWeight(){
return weight;
}

//Mutators
public void setLegs(int legs){
this.legs = legs;
}
public void setWeight(double weight){
this.weight = weight;
}
}

Tunatumia neno this kama variable uliotengeneza ndani ya setter method ina jina sawa na iliopo kwe instance variables za class. Kama hutotumia neno this compiler itapata shida sana kujua ni variable ipi unamanisha, itakupa warning and then itaashum ni variable moja ya kwenye setter.

Kwaio Setters zote ni mutator na Getters zote ni accessors.

Constructors

A constructor is nothing but a special method that initializes some instace variables as soon as an object is created.
So kama unataka pale tu unapotengeneza object iwe na default properties bhac ua tunatumia constructors kufanya hvo.

Constructors zote hua zina majina sawa na jina la class husika na inakua haina return data type.
Look:
public class Animal {

private int legs;
private double weight;

//constructor
public Animal(int l, double w){
legs = l;
weight = w;
}

//Accessors
public int getLegs(){
return legs;
}

public double getWeight(){
return weight;
}

//Mutators
public void setLegs(int legs){
this.legs = legs;
}
public void setWeight(double weight){
this.weight = weight;
}
}

Kwaio ukitengeneza object ya io class lazima utainitilize 'legs' na 'weight', Otherwise, u gonna get a huge error
